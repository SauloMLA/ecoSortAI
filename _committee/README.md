## Propósito

Este comité es una **estructura ejecutiva híbrida** — formal en toma de decisiones, dinámica en ejecución técnica — diseñada para desarrollar un **prototipo físico funcional real** del proyecto **EcoSort IA**.

### Objetivos del prototipo

| # | Objetivo | Criterio mínimo |
|---|----------|-----------------|
| 1 | Clasificación de residuos | 3–5 tipos mínimo |
| 2 | Visión por computadora | Modelo funcional con métricas reales |
| 3 | Hardware integrado | Sistema embebido estable |
| 4 | Mecanismo de desvío | Automático y sin atascos |
| 5 | Feedback visual | Dashboard funcional |
| 6 | Evaluación | Métricas reales documentadas |
| 7 | Defensa académica | Proyecto defendible ante jurado |

---

## Modelo de gobernanza

### Componente formal
- El **Chair** abre y cierra cada sesión
- Se definen objetivos por sesión
- Se registran decisiones en `DECISION_LOG.md`
- Se asignan responsables
- Se actualiza `CURRENT_STATE.md`

### Componente dinámico
- Los miembros técnicos interactúan libremente dentro de su turno
- Se permiten debates intensos
- Se exige pensamiento crítico
- Se permite iteración rápida
- Se prioriza **ejecución real** sobre teoría excesiva

---

## Relación con el equipo real

Este comité **asesora y estructura** el trabajo de un equipo universitario real de 5 personas. **No lo reemplaza.** El comité identifica dimensiones técnicas que el equipo aún no ha considerado y propone expertos adicionales cuando es necesario.
