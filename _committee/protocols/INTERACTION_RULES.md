# Reglas de Interacción — Comité EcoSort IA

> Estas reglas gobiernan cómo se comunican los miembros del comité durante las sesiones.

---

## 1. Anuncio obligatorio

Todo miembro **debe identificarse** antes de hablar:

```
"Soy [rol], y mi análisis es..."
"Soy [rol], del subcomité [nombre]. Mi evaluación es..."
```

### Ejemplo
```
"Soy ML Researcher del subcomité AI & CV. Mi análisis sobre la selección 
de modelo es que MobileNetV2 ofrece el mejor balance entre precisión y 
latencia para nuestro hardware objetivo."
```

## 2. Declaración de investigación

Si un miembro necesita investigar antes de opinar, debe declararlo:

```
"Soy [rol], estoy investigando [tema] para evaluar [objetivo]."
```

### Ejemplo
```
"Soy Edge Inference Specialist. Estoy investigando el rendimiento de 
TFLite en Raspberry Pi 4 para evaluar si cumple con el requisito 
de latencia de 3 segundos."
```

## 3. Relevos explícitos

Al terminar su intervención, un miembro puede ceder la palabra:

```
"Cedo la palabra a [rol] para profundizar en [tema]."
```

### Ejemplo
```
"Cedo la palabra a Model Optimization Engineer para que evalúe las 
opciones de quantización del modelo que propongo."
```

## 4. Posturas obligatorias

En toda discusión sustantiva, **siempre debe existir**:

| Elemento | Responsable | Obligatoriedad |
|----------|-------------|----------------|
| Al menos una postura crítica | Cualquier miembro / Devil's Advocate | OBLIGATORIO |
| Evaluación de riesgos | Technical Director / FMEA Analyst | OBLIGATORIO |
| Métricas sugeridas | Evaluation Scientist / Impact Measurement Expert | OBLIGATORIO |

## 5. Formato de debate

### Estructura de intervención
```
1. Anuncio: "Soy [rol]..."
2. Posición: "Mi análisis/evaluación/propuesta es..."
3. Evidencia: "Esto se basa en [datos/investigación/experiencia]..."
4. Riesgo: "El riesgo de esta postura es..."
5. Recomendación: "Recomiendo [acción]..."
6. Relevo (opcional): "Cedo la palabra a..."
```

### Estructura de contraargumento
```
1. Anuncio: "Soy [rol]..."
2. Referencia: "Respecto a lo que dijo [rol]..."
3. Desacuerdo: "Considero que [punto alternativo]..."
4. Evidencia: "Porque [datos/argumento]..."
5. Propuesta: "Propongo en cambio [alternativa]..."
```

## 6. Prohibiciones

- **NO** hablar sin anunciarse
- **NO** hacer afirmaciones sin fundamentación
- **NO** ignorar riesgos o posturas críticas
- **NO** interrumpir (excepto el Chair por razones de foco)
- **NO** proponer sin métricas de éxito
- **NO** cerrar un tema sin asignar responsable

## 7. Escalamiento

Si hay desacuerdo irreconciliable entre miembros:

1. El Chair solicita a ambas partes que presenten evidencia
2. El Technical Director evalúa viabilidad técnica
3. El Product Strategist evalúa impacto en el MVP
4. El Chair toma la decisión y la registra con justificación
