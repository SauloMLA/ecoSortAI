# Core Team — Equipo Universitario Real

> Este equipo está compuesto por **5 personas reales**. El comité los asesora y estructura su trabajo, pero **no los reemplaza**.

---

## Composición

| # | Rol en equipo | Rol en comité | Perfil |
|---|---------------|---------------|--------|
| 1 | Programador | Software Lead + Integrador IA | `SOFTWARE_LEAD.md` |
| 2 | Diseñador 3D | Mechanical Lead | `MECHANICAL_LEAD.md` |
| 3 | Constructor | Integrador físico | `PHYSICAL_INTEGRATOR.md` |
| 4 | Documentación | Operaciones + Historiador | `DOCUMENTATION_OPS.md` |
| 5 | Miembro libre | QA Interno + Red Team local | `QA_RED_TEAM.md` |

## Principios de relación comité ↔ equipo

1. **El comité propone, el equipo dispone** — Las recomendaciones son eso: recomendaciones
2. **Tareas accionables** — Toda asignación debe ser concreta, medible, y realizable
3. **Respeto por los recursos** — El equipo tiene tiempo, presupuesto y habilidades limitadas
4. **Feedback bidireccional** — El equipo puede cuestionar recomendaciones del comité
5. **Documentación** — Toda asignación se registra formalmente

## Flujo de trabajo

```
Chair → define objetivo de sesión
  └→ Subcomités → analizan y proponen
       └→ Product Strategist → valida alcance
            └→ Technical Director → valida viabilidad
                 └→ Chair → asigna al Core Team
                      └→ Core Team → ejecuta y reporta
```
