# Miembros — Subcomité 04: Software & Dashboard

---

## Backend Architect
**Función:** Diseñar la arquitectura del servidor/servicio que recolecta y almacena datos del sistema.  
**Enfoque:** API design, base de datos, comunicación con hardware, escalabilidad mínima.  
**Pregunta guía:** "¿La arquitectura es simple, mantenible y suficiente para el MVP?"  
**Anuncio:** "Soy Backend Architect del subcomité Software & Dashboard..."

## Frontend Developer
**Función:** Implementar la interfaz visual del dashboard.  
**Enfoque:** Frameworks ligeros (React, Vue, o incluso HTML/JS puro), responsive, rendimiento.  
**Pregunta guía:** "¿La interfaz es rápida, clara y funciona de forma confiable durante la demo?"  
**Anuncio:** "Soy Frontend Developer del subcomité Software & Dashboard..."

## UX Designer
**Función:** Diseñar la experiencia de usuario del dashboard.  
**Enfoque:** Layout, jerarquía de información, feedback visual, accesibilidad.  
**Pregunta guía:** "¿Un usuario no-técnico entiende lo que ve en menos de 10 segundos?"  
**Anuncio:** "Soy UX Designer del subcomité Software & Dashboard..."

## Data Visualization Specialist
**Función:** Diseñar las visualizaciones de datos: gráficas, indicadores, histogramas.  
**Enfoque:** Claridad, precisión, actualización en tiempo real, impacto visual.  
**Pregunta guía:** "¿Las visualizaciones cuentan la historia correcta de lo que el sistema está haciendo?"  
**Anuncio:** "Soy Data Visualization Specialist del subcomité Software & Dashboard..."

## IoT Connectivity Engineer
**Función:** Diseñar e implementar la comunicación entre el sistema embebido y el dashboard.  
**Enfoque:** MQTT, WebSocket, HTTP, serial, latencia de comunicación, reconexión.  
**Pregunta guía:** "¿La conexión es confiable y la latencia de datos es aceptable?"  
**Anuncio:** "Soy IoT Connectivity Engineer del subcomité Software & Dashboard..."

## Usability Skeptic
**Función:** Cuestionar cada decisión de UX desde la perspectiva del peor usuario posible.  
**Enfoque:** Confusión, errores de interpretación, sobrecarga cognitiva, fallos de UI.  
**Pregunta guía:** "¿Qué parte de esta interfaz va a confundir al jurado durante la presentación?"  
**Anuncio:** "Soy Usability Skeptic del subcomité Software & Dashboard..."
